§ex.7 Genetic Algorithms

遺伝的アルゴリズム

Genetic Algorithms

遺伝的アルゴリズム(Genetic Algorithms)とは、遺伝と、(最)適者生存による自然淘汰の仕組みを模すことで、複雑な問題に対する最適解を探索する手法のことです。

そのため、生物学の用語を倣って、ある命題に対する解の候補を遺伝子(gene)、解(遺伝子)の集合体を個体(individual)と呼びます。

適応度の高い個体を優先して交叉(crossover)や突然変異(mutation)などの操作を繰り返すことで、最適解の探索を行います。

例えばN700系新幹線や人工衛星のアンテナの形状は、多数のパターンのデザインをシミュレーションによって算出された適応度をもとに遺伝的アルゴリズムで最適化されています。

特徴

features

遺伝的アルゴリズムには、評価関数の可微分性や単峰性などの知識がない場合であっても適用が可能というメリットがあります。

この性質を利用して、組合せ爆発が起こってしまうNP困難な問題に対して、実行可能時間内で最適な解を求めることができる手法として注目されていますが、一般に大域的最適解に収束する保証はありません。

また、どんな問題にでも使えるわけではなく、評価関数が全順序性であることと、探索空間が位相(topology)を持つことが必要です。

アルゴリズム

Algorithms

ある世代の全個体数を $N$ とおきます。

  1. あらかじめ $N$ 個の個体が入る集合を二つ用意します。ここで、この二つの集合を「現世代」、「次世代」と呼ぶことにします。
  2. 現世代に $N$ 個の個体をランダムに生成します。
  3. 評価関数を用いて、現世代の各個体の適応度をそれぞれ計算します。
  1. ある確率で次の三つの動作のいずれかを行い、その結果を次世代に保存します。
    • 個体を二つ選択して交叉を行う。
    • 個体を一つ選択して突然変異を行う。
    • 個体を一つ選択してそのままコピーする
  2. 次世代の個体数が $N$ 個になるまで上記の動作を繰り返します。
  3. 次世代の個体数が $N$ 個になったら、次世代の内容を全て現世代に移します。
  4. ある収束条件を満たしたら終了し、最終的な世代の中で最も適応度の高い個体を「解」として出力します。

収束判定条件

convergence criteria

先のアルゴリズムの収束条件はいくつか考えられており、主に以下の四つが挙げられます。

  • 集団内の最大適応度が、ある閾値より大きくなった。
  • 集団全体の平均適応度が、ある閾値より大きくなった。
  • 集団の適応度増加率が、ある閾値以下の世代が一定期間続いた。
  • 世代交代の回数が、規定の最大世代数 $G$ に達した。

    • エンコーディング

    encoding

    最適化問題を遺伝的アルゴリズムで解くためには、解の特徴を遺伝子として数式的に表現する必要があります。

    その方法もいくつか考えられており、主に次の四つが挙げられます。

    • バイナリ 各遺伝子を、$0,1$ のビッドとして表現する符号化法のこと。ナップザック問題などで用いられる。
    • 順序 各順序を順序を示す数字として表現する符号化法のこと。巡回セールスマン問題や仕事の順序最適化などの並べ替え問題の表現に用いることができる。
    • 実数値 各遺伝子を数値(または文字)として表現する符号化法のこと。バイナリエンコーディングでは表現が難しい場合に用いられる。
    • 木構造 配列ではなく、木構造データを用いて表現する符号化法のこと。遺伝的プログラミングや進化的プログラミングなどで用いる。

    Genetic Operations

    ここまで、符号化法で定式化して、アルゴリズムを回し、収束を判定するところまで説明しました。

    ここから、アルゴリズムの中心的役割を果たす繁殖(reproduction)の流れ、つまり選択(selection)、交叉(crossover)、突然変異(mutation)を見ていきます。

    選択

    selection

    選択は、適応度に基づいて個体を確率的に増やしたり減らしたりするための対象を選び出す操作のことです。

    生物の自然淘汰をモデル化したもので、主に次の四つが挙げられます。

    • ルーレット 各個体の適応度がルーレット盤の面積と比例させる手法。適応度が正であることが前提であり、個体間の適応度の差が大きい場合は、局所最適解に陥りやすい。
    • ランキング $n$ 番目に適応度が高い場合は確率 $p_n$ というように、順位に応じて予め決められた確率を割り振る方法。適応度の差がない個体間にも差がついてしまう。
    • トーナメント 予め決めた個体数をサンプリングし、その中で最も適応度が高い個体を抽出する手法。トーナメントサイズによって選択圧を調整することが可能。
    • エリート 適応度が高い順に予め決められた数の個体を選び、それらをそのまま選択する手法。世代間で適応度の最大値が下がらないことが保証されるが、解の多様性が失われる。

    交叉

    crossover

    交叉とは、選ばれた二つの個体の遺伝子の一部を入れ替える操作のことです。

    生物の交配をモデル化したもので、通常 $80$〜$95$％の確率で発生するように設定します。

    基本的な交叉モデルとして、次の四つが挙げられますが、順列エンコーディングに対応するためには別の手法が必要です。

    • 一点交叉 遺伝子が交叉する場所(交叉点)をランダムに決定し、その場所より後ろの遺伝子を入れ替える手法。効率が低いので、現在はあまり使われていません。
    • 二点交叉 二つの交叉点をランダムに決定し、その間の遺伝子を入れ替える手法。
    • 多点交叉 三つ以上の交叉点を持つ手法。ただし、上記二つの手法より良い値が得られることがほとんどないので、あまり用いられていない。
    • 一様交叉 各遺伝子のそれぞれを所定の確率(通常は $1/2$ )で入れ替える手法。二点交叉が得意とする問題を苦手とし、逆の性質を示す。

    交叉

    順列エンコーディング対応

    順列エンコーディングをしている場合、交叉が満たさないといけない条件は、同じ遺伝子が重複しないことです。そこで、次にあげるような八つの手法が主に提案されています。

    • 循環交叉 双方の親から重複しない組み合わせ $(1,3,4)$ を選んで、遺伝子の一部ずつを受け継ぐ手法。(ex. $X31X4$ と $Y14Y3$ を交叉させて $X14X3$ と $Y31Y4$ を作る)
    • 順序交叉 片方の親からは一部をそのまま受け継ぎ、残りの部分についてはもう片方の親の順番を保存しつつ受け継ぐ手法。
    • 一様順序交叉 片方の親から遺伝子の集合をランダムに選び、その部分の順序のみをもう片方の親の順序に従って並び替える手法。
    • 一様位置交叉 双方の親から同遺伝子座の遺伝子対を複数選び、その部分だけ入れ替える手法。
    • 部分写像交叉 片方の親 $P_1$ からは一部をそのまま受け継ぎ、残りの部分に関してはもう一方の親 $P_2$ からそのまま受け継ぐ手法。なお、この時重複する遺伝子 $x$ については、$P_1$ 上の $x$ と同遺伝子座に存在する $P_2$ 上の遺伝子を受け継ぐ。

    交叉

    順列エンコーディング対応

    • サブツアー交換交叉 双方の親間で、ある共通する数字の組み合わせからなる部分配列 $X_1,X_2$ が存在する場合(遺伝子座、順番は問わない)に、それらを交換したもの、順番を逆にした $\overline{X_1},\overline{X_2}$ の状態で入れ替えたものの合計四個の子を生成する手法。
    • 辺組換え交叉 順列の繋がり(辺)に着目した交叉手法。双方の親が共通して保有する辺をなるべく多く受け継ぐように子を生成する。
    • 枝組立て交叉 両親のいずれかが保有する辺のみを用いて複数のサブツアーに分割された不完全解を生成し、そのサブツアー間を、なるべく短い辺を用いて接続し直すことで完全解を含む子を生成する手法。

    ちなみに、この他にも以下に挙げるような様々な手法が提案されています。

  • 算術交叉
  • 平均化交叉
  • ブレンド交叉
  • 単峰性正規分布交叉
  • シンプレックス交叉

    • 突然変異

    mutation

    突然変異は、個体の遺伝子の一部をランダムに変化させることで、生物における突然変異をそのままモデル化したものです。

    遺伝的アルゴリズムにおける突然変異の意義は、局所解に陥ることを防ぐことで、通常発生する確率は $0.1$％〜$1$％、高くても数％と設定されます。

    確率が低すぎると局所解に陥りやすく、高すぎると解が収束しにくくなる、といったトレードオフの関係にあります。

    主要なものには、次に挙げる九つがあります。

    • 置換 ランダムに選ばれた遺伝子を対立遺伝子(バイナリエンコーディングの場合)に置き換える手法。
    • 摂動 ランダムに選ばれた遺伝子の値に微小量を加算(減算)する手法。
    • 交換 ランダムに選ばれた二つの遺伝子を入れ替える手法。
    • 逆位 ランダムに選ばれた二点の遺伝子間を逆順に並べ替える手法。

    突然変異

    mutation

    • 撹拌 ランダムに選ばれた二点の遺伝子間をランダムに並べ替える手法。
    • 転座 ランダムに選ばれた二点の遺伝子間を別の遺伝子座に移動する手法。
    • 重複 ランダムに選ばれた二点の遺伝子間を別の遺伝子座に複製する手法。※ 遺伝子長が変化する。
    • 挿入 ランダムに選ばれた場所に任意の遺伝子を挿入する手法。
    • 欠失 ランダムに選ばれた遺伝子を削除する手法。

    繁殖

    離散世代モデル

    繁殖の仕方にもいくつか種類があります。

    離散世代モデル(discrete generation model)は、全ての個体が一斉に繁殖を行い、現世代集合全体が次世代集合に置き換えられるモデルです。

    従って、世代間における個体のオーバーラップが存在しません。

    連続世代モデル

    連続世代モデル(continuous generation model)は、現在の世代のうち一定の割合の個体だけを入れ替え、残りはそのまま次世代に残すモデルです。

    この時、各世代の個体のうち、次世代と入れ替えるものの割合を世代ギャップ(generation gap)と呼び、世代間最小ギャップモデル(minimum generation gap)などの手法が考察されています。

    ちなみにこの手法はランダムに選択された親個体二個とそこから作られる子個体二個を一つの家族とし、その中でエリート選択とルーレット選択を行うことで、次世代に残す二個体を選ぶ手法です。

    Common problem

    初期収束

    最初の方の世代で「偶然」他の個体よりも適応度が圧倒的に高い個体が生じた場合、その個体の遺伝子が集中的に爆発的に増え、探索がかなり早い段階で収束してしまうことがあります。

    その場合は、パラメータを修正するなどして繰り返し試行を繰り返す必要があります。

    なお、次の手法によって初期収束を防ぐこともあります;「集団数の増加 / トーナメントサイズの縮小 / 突然変異率の増加 / より効果的な突然変異操作の導入

    大変異

    大変異は、予め定めておいた条件に応じて、特定の世代においてのみ突然変異率を急激に高める手法です。

    これは、自然界における環境の激変を模倣したものです。

    適応変異

    親個体間の不一致度合い(ハミング距離)に応じて、生成された子個体の突然変異率を変化させることです。

    遺伝子の異なる親同士の子供は突然変異が起きにくく、逆に似た親同士の子供は突然変異が起きやすいように調整します。

    まとめ

    summary

    遺伝的アルゴリズム(Genetic Algorithms)とは、生物の遺伝を模したアルゴリズムでした。

    (最)適者生存による自然淘汰によって、最適解を探索することができる手法です。

    対象となる問題やその評価関数に関する数学的な知識がない場合でも適用が可能でした。
